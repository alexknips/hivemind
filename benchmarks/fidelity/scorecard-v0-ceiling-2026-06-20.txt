WHAT THE SCHEMA CAN CAPTURE (P=R=F1=1.00 at ceiling):
  Nodes: Decision, DecisionRequest, Evidence, Hypothesis, Blocker, Option
  Edges: HAS_OPTION, CHOSE

SCHEMA GAPS (structural FN, ceiling F1=0.00):
  Actor nodes (6 FN): no "actor" CaptureItem kind — actors referenced only
    via actor_id/accepted_by/rejected_by/blocked_actor_id fields, never as
    standalone nodes in produced_graph. Affects D1, D2, D3, E2.
  BASED_ON (9 FN): biggest single gap — evidence-to-decision links not in
    CaptureItem schema. All evidence-backed decisions lose this edge.
  SUPERSEDES (4 FN): supersedes_id is in CaptureItem but NOT wired in
    produced_graph (comment: "NOTE: BASED_ON, SUPERSEDES ... NOT in schema").
    Affects B-archetype cases (B1, B2, B3).
  ASSUMES (4 FN): assumes_ids in schema, not in produced_graph. Affects C, E3.
  REFUTES (2 FN), SUPPORTS (2 FN): same gap.
  Actor-linkage edges (8 FN total):
    ACCEPTED_BY (1), PROPOSED_BY (1), REJECTED_BY (1),
    BLOCKER_FOR_DECISION (2), BLOCKER_REQUIRED_OWNER (1),
    DECISION_REQUESTED_BY (2) — all actor-relational edges require Actor
    nodes that can't be produced.

ANTI-NOISE CASES (precision check):
  F1 (unilateral-no-alternatives): macro-F1=1.00 ✓ — bare Decision, no FP
  F2 (status-update-non-decision): macro-F1=1.00 ✓ — correctly empty
  F3 (request-only): macro-F1=1.00 ✓ — DecisionRequest only, no FP Decision

PATH TO IMPROVE macro-F1 (schema changes, not LLM changes):
  1. Add "actor" CaptureItem kind (closes Actor + actor-linkage gaps) → +~0.05
  2. Wire supersedes_id → SUPERSEDES edge in produced_graph → +~0.03
  3. Wire assumes_ids → ASSUMES edges in produced_graph → +~0.03
  4. Wire supports_ids/refutes_ids → edges → +~0.02
  5. Add evidence_id references to CaptureItem → BASED_ON edges → +~0.06
