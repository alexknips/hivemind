Key Findings — PRELIMINARY
============================================================

1. THE CURVE DOES NOT RISE WITH DISPERSION.
   The saturation finding from Phase 1 (T3/T5 Δ=0 at G0) extends across
   ALL grades G1–G3 for binary tasks. claude-sonnet-4-6 detects contested
   status and refuted assumptions from dispersed, unlabeled documents
   without "Status: Contested" or "Refuted" headers, within a single
   context window. Capture-time linking provides no measurable binary-task
   uplift in the Tier 1.5 corpus.

2. BINARY SATURATION AT ALL GRADES (T3, T5): Δ = 0.000.
   Control scores 1.000 at every grade — the model does cross-document
   linking in context. Example (T5G2 sharding): control response correctly
   identifies that INC-882's partition imbalance refutes the architecture
   decision's uniform-distribution assumption, even across two unlinked
   documents. Example (T3G3 data-retention): model correctly connects
   7-year legal requirement, 365-day engineering proposal, and 60-month
   SLA across three documents with no cross-references, using numerical
   inference (365 days << 7 years).

3. TOKEN-F1 UPLIFT EXISTS BUT IS FORMAT-DRIVEN, NOT LINKING-DRIVEN.
   Treatment JSON elicits more concise, structured responses (fewer
   spurious tokens → higher precision). Δ values:
     T1G1 (decision-recall, G1): +0.121
     T4G1 (onboarding, G1):      +0.108
     T1G2 (decision-recall, G2): +0.171
   These match the Phase 1 T1/T2/T4 pattern (+0.069–+0.181). Both arms
   have full access to the same information; treatment wins on response
   format precision, not on resolving information gaps.

4. G2 T1G2 SHOWS HIGHEST SINGLE ΔCASE (+0.171): The 2-doc control forces
   cross-doc synthesis that adds hedging tokens ("evidenced by the Loki
   v2.9 installation in the setup notes"), diluting token-F1 precision.
   Treatment JSON enables a direct response. This is a verbosity effect,
   not a linking effect.

5. GOLD-TEXT DESIGN ISSUE IN T4G1: word-form numbers ("five", "sixty")
   in gold suppress scores on both arms equally (source uses "5%", "60%").
   Affects absolute score but not relative Δ. Recommend fixing in
   corpus revision.

============================================================
Methodological Limitation — What Tier 1.5 Cannot Test
============================================================

The expected "capture-time linking" uplift for binary detection tasks
requires a retrieval scenario where documents CANNOT co-present in a
single context window. In Tier 1.5, all documents fit in one call.

The real hypothesis is:
  HiveMind's pre-linked graph enables cross-document retrieval that
  a flat document store cannot — when querying across a corpus too
  large for a single context.

This test is not possible with a context-window-size corpus. A valid
test would require:
  - A corpus of hundreds of decisions across dozens of documents
  - A retrieval step that selects k documents from the full corpus
  - Comparison: flat retrieval (BM25/embedding) vs. graph-aware retrieval
    (following ASSUMES/REFUTES/SUPERSEDES links from the query entry point)
  - Control arm: flat retrieval might miss the refuting document
  - Treatment arm: graph walk from the decision node reaches the refutation

Tier 1.5 tests response quality given equal information access.
Phase 2 (with Challenger corpus and real org ADR archives) remains the
right venue for testing retrieval-dependent uplift.
