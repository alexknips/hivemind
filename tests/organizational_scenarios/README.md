# Organizational Scenario Fixtures

These fixtures are the compact real-world acceptance suite for mixed human and agent decision memory.

- `production-incident-response` proves evidence-driven mitigation, refuted hypotheses, and supersession from a contested hotfix path to rollback.
- `remote-db-architecture-choice` proves non-developer requirements, security review evidence, and supersession from embedded prototype storage to service-owned writes.
- `product-launch-readiness` proves contested readiness, rejected launch-now status, legal/support context, and accepted phased rollout.
- `security-vulnerability-triage` proves severity reassessment after exploit evidence, hypothesis refutation, customer/legal coordination, and disclosure disagreement.
- `hiring-capacity-planning` proves non-code organizational memory, capacity assumptions, budget evidence, rejected deferral, and accepted full-time hiring.

Every scenario includes at least one human actor, one agent actor, decisions, evidence, options, hypotheses, and explicit relation events. The integration test projects the scenarios into the memory graph and checks status derivation, evidence and hypothesis context, supersession chains, and topic/status filtering.
